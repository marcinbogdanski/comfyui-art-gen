## Big Love Prompting Lessons

- Big Love works best with detailed text plus matching reference images.
  References alone are weak, and unusual poses/contact points are fragile.
- Image 2 in the reference workflow is a real second reference when enabled,
  not generic guidance. Extra references should be possible by chaining more
  Reference Conditioning, but they can conflict or dilute control.

## Big Love Txt2Img/Upscale Workflow

- Added a second Big Love Klein 2 txt2img/upscale Civitai workflow. Workflow
  source, saved paths, and node-pack revisions are documented in
  `models/big-love-klein2-txt2img-upscale-workflow.md`.
- This workflow brought in `rgthree-comfy`, `ComfyUI-Inspire-Pack`, and
  `ComfyUI-Impact-Pack`. Important dependency chain: `KSampler //Inspire`
  loads from Inspire Pack but calls Impact Pack at runtime.
- Added `/mnt/data/comfyui/custom_nodes/skip_download_model` so Impact Pack
  does not auto-download SAM model weights unless SAM/Impact nodes are
  intentionally used later.
- rgthree warns that ComfyUI's Node 2.0 rendering can break rgthree nodes. If
  rgthree nodes show as missing or lose properties, disable Node 2.0/new node
  rendering and hard-refresh.
- If the Inspire sampler/frontend serialization breaks, a reasonable workaround
  is replacing `KSampler //Inspire` with core `KSampler` using the same visible
  sampling settings. For batch diversity, avoid fixed seed behavior; use
  randomize/incremental seeds or queue separate runs.
