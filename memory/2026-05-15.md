# 2026-05-15

## HunyuanImage 3.0 Research

- Checked current HunyuanImage 3.0 options after user asked about text-to-image
  and image-plus-text-to-image workflows, specifically excluding HunyuanImage
  2.1.
- Official Tencent `HunyuanImage-3.0` supports text-to-image; the Instruct and
  Instruct-Distil variants add image editing / image-conditioned generation
  features, but official requirements are far above this host's single 24 GB GPU
  target.
- Credible community ComfyUI direction appears to be EricRollei's
  `Comfy_HunyuanImage3` plus quantized HF checkpoints. The most plausible local
  experiment is `HunyuanImage-3.0-Instruct-Distil-NF4-v2`, but it is still risky
  on this machine because advertised 48 GB GPU suitability does not necessarily
  map cleanly onto two 24 GB GPUs.
- Clarified architecture: HunyuanImage 3.0 is described as a native multimodal
  autoregressive / MoE framework, but image synthesis still uses a diffusion
  sampling stage. The Distil NF4 note about "only 8 diffusion steps" refers to
  the denoising sampler, not pixel-by-pixel autoregressive decoding.
