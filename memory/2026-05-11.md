# 2026-05-11

## ComfyUI Server Bring-Up

- Set up this repo as a lightweight Docker-managed ComfyUI server for the
  headless `x399` GPU machine.
- Chose manual `docker run --rm -it` as the operating model instead of
  Docker Compose, because Marcin wanted explicit foreground runs rather than
  a background service.
- Added a Dockerfile based on the official PyTorch CUDA 13 runtime image,
  with ComfyUI installed inside the image and no CUDA/PyTorch/ComfyUI
  dependencies installed on the host.
- Added `run_comfyui.sh` to create writable bind-mount directories under
  `/mnt/data/comfyui`, run on GPU 0 by default, publish port `8188`, and
  always pass `--listen 0.0.0.0` so browser access works from another
  machine.
- Confirmed the server is intended to be reached at `http://x399.lan:8188/`
  and should not be exposed directly to the public internet.

## First Working Model

- Downloaded and tested the single-file Flux.1 Schnell FP8 checkpoint in
  `/mnt/data/comfyui/models/checkpoints`.
- Documented the working model download command and exact raw workflow PNG
  URL in `models/flux-schnell-fp8.md`.
- Captured the loading gotcha: if ComfyUI asks for
  `v1-5-pruned-emaonly-fp16.safetensors`, the Flux workflow did not load and
  the default SD1.5 workflow is still active.

## Repo Notes

- Added a short `README.md` with build, run, data, and access instructions.
- Added `AGENTS.md` with operational preferences for future agents:
  no host CUDA/toolkit installs, keep large model files out of git, use
  `/mnt/data/comfyui` for persistent data, and prefer one GPU per ComfyUI
  instance.
